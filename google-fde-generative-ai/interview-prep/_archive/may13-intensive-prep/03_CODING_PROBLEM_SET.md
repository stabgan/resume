# Coding Problem Set - Must 60, Should 40, Stretch 50

This is the aggressive coding track. It is compatible with Blind 75 and NeetCode 150, but reordered for Google/FDE risk.

## How To Solve Every Problem

No execution. Plain editor.

1. Restate.
2. Clarify constraints.
3. State brute force.
4. State optimized invariant.
5. Code.
6. Dry-run normal case.
7. Dry-run edge case.
8. Complexity.
9. Tests.

If you cannot solve in 25 minutes:

- Read only the pattern hint.
- Close the hint.
- Re-code.
- Mark red for redo in 24 hours.

## What Makes A Strong Google Coding Answer

Strong answer:

- Clarifies before coding.
- Names invariant.
- Codes simple readable Python.
- Dry-runs edge case.
- Discusses complexity.
- Mentions production test cases.

Weak answer:

- Silent typing.
- Fancy code that cannot be dry-run.
- Optimizes before correctness.
- Cannot explain edge cases.
- Depends on running the interpreter.

## Redo System

Mark each problem:

- Green: solved under 30 min, dry-run clean.
- Yellow: solved with hint or bug.
- Red: failed, pattern unclear, or code not dry-runnable.

Redo:

- Yellow after 24 hours.
- Red after 12 hours and again after 48 hours.

You should enter May 13 with no red Must 60 problems.
