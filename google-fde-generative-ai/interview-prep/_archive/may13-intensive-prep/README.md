## What Changed From The Rapid Pack

- DSA is no longer "one problem a day." It is a high-volume pattern rebuild.
- System design and ML system design are first-class tracks.
- Cloud infrastructure is deeper: containers, inference pipelines, SSO, VPC, APIs, migration, data perimeters.
- FDE prep includes decomposition, customer scenarios, presentation defense, and values.
- The coding set is now tiered: **Must 60**, **Should 40**, **Stretch 50**, broadly compatible with NeetCode 150 but biased to Google/FDE.
- Every day has mocks and recall, because knowledge without timed retrieval will not survive the interview.

## North Star

By May 13, you should be able to do all of this cold:

- Solve Google-medium Python problems without running code.
- Explain every core data structure and algorithm pattern.
- Design a classic distributed system with scale, reliability, consistency, and cost tradeoffs.
- Design ML systems: recommendation, ranking, fraud, search, feature store, training/serving, monitoring.
- Design GenAI systems: RAG, agentic workflows, MCP tools, evals, safety, observability, cost/latency.
- Defend Google Cloud deployment choices: Agent Engine vs Cloud Run vs GKE, IAP, VPC-SC, PSC, Apigee, IAM/OAuth/OIDC.
- Tell your top stories with FDE framing: customer pain, technical action, production result, reusable lesson.

## The Interview Shape

Official recruiter PDF:

- `RRK`, 60 minutes.
- `Coding`, 60 minutes.
- Coding: Python/OOP, 30-50 lines, no execution.

External research says adjacent Google Cloud/FDE/CE loops often probe:

- RRK depth: containers, inference pipelines, infrastructure differences.
- Cloud migration and customer engineering scenarios.
- GCA/Googleyness-style hypotheticals.
- Presentation or solution deep dive in longer loops.
- Medium coding with graphs, arrays/strings, hash maps, trees, DP, and OOP design.

So this folder prepares for the official loop and the harder adjacent probes.

## Daily Non-Negotiables

- 6 hours sleep minimum.
- Every coding problem in a plain editor/no execution.
- Every system design answer starts with requirements and constraints.
- Every ML/GenAI design includes evals, monitoring, safety, and cost.
- Every FDE answer ends with customer impact and reusable product feedback.
